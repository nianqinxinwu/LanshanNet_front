```

✅ GET /api/xiluxc.im/history - 获取聊天记录
✅ GET /api/xiluxc.im/user_info - 获取用户信息
✅ POST /api/xiluxc.im/send - 发送消息
```

### 2. 修改商品详情页面

**文件**: `/pages/jj/jj_product_detail/jj_product_detail.vue`

#### 修改1: 添加工厂用户ID字段

在 `mapProduct` 方法中添加工厂相关字段：

```javascript
mapProduct(item) {
    return {
        // ... 其他字段
        factoryId: item.factory_id || item.factoryId || 0,
        factoryUserId: (item.factory && item.factory.user_id) || item.factoryUserId || 0,
        factoryName: (item.factory && item.factory.company_name) || item.factoryName || '',
        // ... 其他字段
    };
}
```

#### 修改2: 实现联系厂家功能

替换 `onContact()` 方法：

```javascript
onContact() {
    // 检查是否有工厂用户ID
    if (!this.detail.factoryUserId) {
        uni.showToast({ title: '无法联系该厂家', icon: 'none' });
        return;
    }

    // 跳转到IM聊天页面
    uni.navigateTo({
        url: `/pages/im/im_chat/im_chat?to_user_type=user&to_user_id=${this.detail.factoryUserId}&nickname=${encodeURIComponent(this.detail.factoryName || '工厂')}`
    });
}
```

## 🔧 技术实现

### IM聊天页面参数

聊天页面 (`im_chat.vue`) 接受以下参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `conversation_id` | String | 否 | 会话ID（如果没有会自动创建） |
| `to_user_type` | String | 是 | 对方用户类型（'user' 或 'admin'） |
| `to_user_id` | Number | 是 | 对方用户ID |
| `nickname` | String | 否 | 对方昵称（用于导航栏显示） |

### 跳转逻辑

1. **获取工厂用户ID**: 从商品详情的 `factory.user_id` 字段获取
2. **参数编码**: 使用 `encodeURIComponent` 编码工厂名称，防止中文乱码
3. **页面跳转**: 使用 `uni.navigateTo` 跳转到聊天页面
4. **会话创建**: 聊天页面会自动调用后端 `start_conversation` 接口创建或获取会话

## 📊 数据流程

```
商品详情页
  ↓ 点击"联系厂家"
获取工厂用户ID (factoryUserId)
  ↓
跳转到IM聊天页面
  ↓ 传参: to_user_type=user, to_user_id=XX, nickname=工厂名
聊天页面初始化
  ↓ onLoad
调用 start_conversation API
  ↓ POST /api/xiluxc.im/start_conversation
获取或创建会话
  ↓ 返回 conversation_id
加载聊天历史
  ↓ GET /api/xiluxc.im/history
显示聊天界面
  ↓
用户可以发送消息给工厂
```

## ⚠️ 注意事项

### 1. 数据依赖

确保商品详情接口返回包含工厂信息：
```json
{
  "data": {
    "id": 1,
    "name": "商品名称",
    "factory": {
      "id": 10,
      "user_id": 100,  // 工厂用户ID（必需）
      "company_name": "工厂名称"
    }
  }
}
```

### 2. 权限检查

- 确保居间人已登录
- 确保工厂用户存在
- 后端会验证会话权限
